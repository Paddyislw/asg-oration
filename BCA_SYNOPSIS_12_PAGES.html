1. INTRODUCTION AND OBJECTIVES OF THE PROJECT

1.1 Introduction

ASG-Oration is an AI-powered career counseling platform designed to democratize access to professional career guidance. Traditional counseling is expensive, time-consuming, and geographically limited. This platform leverages Google's Gemini 2.0 Flash AI to provide 24/7 accessible, intelligent career counseling through conversational interactions.

The system addresses career transition planning, resume optimization, interview preparation, salary negotiation, and skill development. It maintains conversation history across sessions, enabling continuous career development tracking.

1.2 Project Objectives

Primary Objectives:

  • Accessibility: Provide 24/7 global access to career counseling, eliminating geographical and financial barriers
  • Personalization: Deliver context-aware advice based on conversation history with session continuity
  • Comprehensive Support: Cover all career development aspects from entry-level to executive positions
  • Technology Excellence: Implement cutting-edge AI with type-safe, scalable architecture and modern UX
  • Security: Ensure data encryption, secure authentication, and role-based access control
  • Performance: Design serverless architecture with optimized queries and efficient caching

3. PROBLEM DEFINITION AND REQUIREMENTS

3.1 Problem Statement

Current Challenges: Traditional career counseling suffers from limited accessibility (geographic constraints, high costs), scalability issues (one counselor serves limited clients), continuity problems (fragmented advice, lost context), and knowledge gaps (limited industry expertise).

Proposed Solution: AI-powered 24/7 global counseling with complete history, vast knowledge base, affordable access, unlimited scalability, and consistent advice.

3.2 Functional Requirements

  • Authentication: Google OAuth registration, secure sessions, isolated user data
  • Session Management: Create/rename/delete sessions, auto-title generation, pagination
  • Chat: AI conversations, context maintenance, Markdown rendering, real-time indicators
  • UI: Responsive design, dark/light themes, toast notifications
  • Data: Message persistence, session metadata, data integrity

3.3 Non-Functional Requirements

  • Performance: Page load <2s, AI response <1s, queries <500ms, 1000+ concurrent users
  • Scalability: Auto-scaling architecture, 10,000+ sessions support
  • Security: HTTPS/TLS, input validation, encrypted tokens (30-day expiry)
  • Reliability: 99.9% uptime, automated backups, graceful error handling

6. SECURITY MECHANISMS

6.1 Authentication & Authorization

  • OAuth 2.0: Industry-standard Google authentication, eliminates password vulnerabilities
  • Session Tokens: JWT signed with NEXTAUTH_SECRET, HTTP-only cookies, SameSite=Strict
  • RBAC: Users access only their own data, database queries filter by userId

6.2 Data Security

  • Encryption: HTTPS/TLS 1.3 in transit, database SSL/TLS, environment variables in secure vault
  • Input Validation: Zod schema validation, Prisma parameterized queries prevent SQL injection
  • XSS Protection: React automatic escaping, Content Security Policy headers

6.3 Application Security

  • Rate Limiting: Vercel Edge Middleware, 100 requests/minute per IP
  • Error Handling: Generic user messages, detailed server logs, no stack trace exposure
  • Dependencies: npm audit scanning, Dependabot alerts, regular updates

6.4 Infrastructure Security

  • Serverless: Vercel DDoS protection, function isolation, automatic patches
  • Environment: Secrets never in Git, encrypted Vercel variables, separate dev/prod
  • Headers: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, HSTS

6.5 Compliance

  • Privacy: GDPR considerations, data export capability, cascade deletion
  • Audit: Critical operations logged, timestamped actions, IP tracking
